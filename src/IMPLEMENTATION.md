# 実装解説

半透明のたまご（殻）の中で、謎のかたまり（中身＝slime）が重力に従って
デロッと流れるインタラクティブ 3D デモの内部実装メモ。
パラメータを触る人・挙動を直す人向けに、構造と物理モデルを細かく解説する。

---

## 全体像

- **ビルド不要の静的サイト**。`index.html` を HTTP(S) 配信して開くだけ。
  ES モジュールを使うので `file://` では動かない。
- **three.js (r128)** は `index.html` の classic script で読み込み、`window.THREE`
  として各モジュールが参照する（バンドラ・npm 依存なし）。
- 物理は自前実装。**Verlet 積分 + 位置ベース拘束（PBD）** で、
  内圧・距離拘束・形状記憶・表面張力（平滑化）・接触・閉じ込めを組み合わせる。

### モジュール構成と依存

```
index.html
  └─ src/main.js            ← エントリ。構築・物理ステップ・ループ
       ├─ src/icosphere.js  ← three.js 非依存の幾何/物理ヘルパ（純関数群）
       ├─ src/scene.js      ← レンダラ/カメラ/ライト/材質/環境
       └─ src/gravity.js    ← 重力入力（ドラッグ + 端末傾き）→ gTarget
  └─ src/styles.css         ← 見た目
```

`icosphere.js` は数値計算だけの純関数で、three.js に依存しない（テスト・再利用しやすい）。
`scene.js` と `gravity.js` がブラウザ/three.js 依存を引き受け、`main.js` が束ねる。

## 座標系の約束

- three.js の右手系。カメラは +z 側から原点を見る（`scene.js`）。
- したがって **+x=右 / +y=上 / +z=手前（視点側） / -z=画面奥**。
- 重力ベクトル `gCur` もこの座標系。既定は `(0, -G, 0)`（真下）。

## 幾何（`icosphere.js` と `main.js` の eggMap）

### アイコスフィア生成 `buildIcosphere(subdiv)`

正二十面体を `subdiv` 回サブディビジョンし、中点を辞書でキャッシュして
**頂点を溶接した（共有した）連結メッシュ**を作る。返り値:

- `dir`   : 各頂点の正規化方向ベクトル（半径1）。Float32Array(count*3)
- `faces` : 三角形（頂点インデックスの配列の配列）
- `index` : 描画用インデックス（Uint16Array）
- `count` : 頂点数

`detail: 4` でおよそ 2562 頂点。大きいほど滑らかだが計算は重くなる。
頂点が溶接されているので、辺バネ・体積・法線が連続して破綻しない。

### たまご形への写像 `eggMap`（`main.js`）

単位球の方向ベクトルを卵形に写す。

```
taper = 1 - 0.20 * uy      // 上(uy>0)ほど細く
x = ux * R * taper
y = uy * R * 1.32          // 縦に伸ばす
z = uz * R * taper
```

殻はこの形に固定（既定は剛体）。中身を閉じ込める `clampToEgg` も
同じ式で「その高さでの内側半径」を求めるので、下が太い卵形にちゃんと収まる。

## 物理モデル（`main.js` の `step()`）

中身は質点（頂点）の集合で、毎ステップ次の順に処理する。

1. **重力の追従**（`gCur` を `gTarget` へ lerp）
2. **内圧 + 重力の力積算**（`accumulatePressure` + `gCur`）
3. **Verlet 積分**（速度＝現在位置−前回位置、減衰 `C_DAMP`）
4. **距離拘束**（`solveEdges`、`C_STIFF` を `ITER` 回）
5. **形状記憶**（重心に再センタリングした元の丸い形へ少し戻す、`C_SHAPE`）
6. **接触**（殻が柔らかい `softness>0` の時だけ）
7. **閉じ込め**（`clampToEgg`：卵の内側へ押し戻す）
8. **表面張力＝平滑化**（タウバン平滑化、`C_SMOOTH`）→ 再度 `clampToEgg`

### 1. 重力の追従

```
gCur += (gTarget - gCur) * 0.2
```

`gTarget`（入力の目標重力）へ毎ステップ 0.2 ずつ近づける一次遅れ。
係数を上げると傾き入力への反応がきびきびする（下げると緩慢）。

### 2. 内圧 `accumulatePressure(p, faces, frc, P)`

各三角形を外向き法線（×面積）方向に押す力を全頂点へ配る。
強さ `P` は体積比から決める:

```
P = C_PRESS * (cRestVol / 現在体積 - 1)
```

体積が縮むと正圧で膨らみ、膨らむと負圧で戻る＝**弾性的な体積保持/表面の張り**。
ハードな体積保持ではないので、伸縮（びよんびよん）はする。
`C_PRESS`（=`CONFIG.slime.pressure`）が大きいほど張ってプルッと丸くなる。

体積 `computeVolume` は符号付き四面体の総和（発散定理）で閉曲面の体積を出す。

### 3. Verlet 積分

```
next = cur + (cur - prev) * C_DAMP + force * dt²
prev = cur
```

速度を陽に持たず、位置の差分で表す数値積分。`C_DAMP` は速度減衰で、
**1 に近いほど減衰が小さく＝よく揺れる**（プルプル）。小さいほど早く止まる。

### 4. 距離拘束 `solveEdges(pos, edges, stiffness, iter)`

各辺（バネ）を rest 長へ近づける位置補正を `iter` 回反復（Gauss-Seidel 的）。
`stiffness` が大きいほど形を保つ（固い）。小さいほど自由に変形して流れる。
辺は `buildEdges` で重複なく作り、生成時の長さを rest（自然長）にする。

### 5. 形状記憶（shape matching の簡易版）

```
重心 m = mean(cPos)
cPos[i] += (m + cRest0[i] - cPos[i]) * C_SHAPE
```

`cRest0` は原点中心の元の丸い形。これを**現在の重心へ平行移動した目標**へ
少し引き戻す（回転は無視）。固体っぽい「丸い形へ戻る弾性」を与える。
`C_SHAPE = 0` で完全に流体寄り（丸へ戻らない）、大きいほど固体寄り。

### 6. 接触（殻が柔らかい時のみ）

`CONFIG.shell.softness > 0` のときだけ有効。中身と殻の頂点が
半径 `CR` 内に近づいたら重なり量だけ押し合う、素朴な球同士の反発。
既定は殻が剛体（softness=0）なので**この処理は走らない**。

### 7. 閉じ込め `clampToEgg()`

中身の各頂点を、その高さ `y` での卵内側の水平半径へ押し戻す安全網。
`eggMap` と同じテーパー式を使い、上下限（`EGG_YLIM`）と
水平半径（`maxR`）でクランプする。殻を突き抜けないための最終防御。

### 8. 表面張力＝タウバン平滑化

`smoothLaplacian(pos, adj, lambda, tmp)` は各頂点を隣接平均へ `lambda` 寄せる
ラプラシアン平滑化。隣接リスト `adj` は `buildAdjacency` で辺から CSR 形式で作る。

そのまま掛けると縮むので、**正の係数で平滑化 → 少し大きい負の係数で押し戻す**
（タウバン平滑化）ことで、全体の形・体積をほぼ保ったまま
高周波の凹凸（しわ）だけ均す。これが表面張力的に滑らかな膜を作る。
`C_SMOOTH`（=`smooth`）が大きいほど滑らか。最後にもう一度 `clampToEgg`。

> 体積の厳密保持（重心まわりの等方スケールで毎フレーム元体積へ戻す処理）は
> **行っていない**。丸み・張りは内圧と平滑化の弾性が担い、体積は伸縮してよい。

## メインループ（`main.js` の `loop()`）

**固定タイムステップ + アキュムレータ**でフレームレート非依存にする。

```
DT = 1/120
acc += min(経過時間, 0.1)
while (acc >= DT && steps < 8) { step(); acc -= DT; steps++; }
```

1 フレームの経過時間を 0.1s で頭打ちにして（タブ復帰時の暴走防止）、
固定 1/120 秒刻みで最大 8 回まで物理を進める。描画前に中身の頂点法線を
再計算（`computeVertexNormals`）。殻は剛体なら法線は初期計算のまま。

### リサイズ（`resize()`）

縦長スマホでも卵が収まるよう、要求縦横サイズとアスペクト比から
必要なカメラ距離を求めて `camera.position.z` を決める。
`resize` と `screen.orientation` の `change` で再計算する。

## 重力入力（`gravity.js`）

`createGravity(canvas, G)` は `gTarget`（目標重力ベクトル）と `update()` を返す。
入力は 2 系統で、**端末傾きが有効になったらドラッグ計算は止まる**。

### PC：ドラッグで傾ける

ポインタのドラッグ量を `tiltX`（前後）/`tiltZ`（左右）に積む（上限あり）。
`update()` で下向きベクトルを X まわり→Z まわりに回して `gTarget` を作る。
前後ドラッグ（`tiltX`）が z 成分を生むので、**PC でも奥行き方向の重力**になる。

### スマホ：端末の傾き `onOrientation`

`deviceorientation` の `beta`（前後/pitch）・`gamma`（左右/roll）から
**端末座標系での下向き単位ベクトル**を正しい射影で求める:

```
gx =  cos(beta) * sin(gamma)   // 左右（右が +x）
gy = -sin(beta)                // 上下（上が +y）
gz = -cos(beta) * cos(gamma)   // 奥行き（手前が +z, 奥が -z）
```

- 単純な `sin(gamma)` 近似だと、端末を立てた時（beta>90°）にジンバルで
  gamma の符号が飛んで**左右が反転**する。`cos(beta)` 込みの上式は連続で反転しない。
- `gz` により**奥行き方向の重力**も出る（寝かせると奥/手前へ流れる）。
- 画面の向き（縦/横 = `screen.orientation.angle`）に応じて、画面平面内の
  `gx, gy` のみ回転補正する（奥行き `gz` は画面法線方向なので不変）。
- 最後に `G` 倍して `gTarget` にセット。

### iOS のモーション許可

iOS Safari は `DeviceOrientationEvent.requestPermission()` を
**ユーザー操作のハンドラ内で同期的に**呼ぶ必要があるため、
`pointerdown` / `click` / `touchend` のいずれでも `enableMotion()` を走らせる。
許可されたら `deviceorientation` を購読、拒否時は次のタップで再要求できるようにする。

## 描画・材質（`scene.js`）

- **レンダラ**: antialias + alpha、ACESFilmic トーンマッピング、sRGB 出力。
- **環境マップ**: 反射に「柄」が出ないよう、4×4 の単色（`#262b34`）を
  `PMREMGenerator` に通したフラットな環境。グラデ環境だと殻に縞が映り込むため。
- **ライト**: 主光は 1 灯のディレクショナルライト。極小の環境光を補助に足す。
- **殻 `shellMat`**: 薄い透明ガラス。`opacity` は `CONFIG.shell.thickness`
  （厚み＝不透明度）。`depthWrite:false` + `renderOrder` で中身が透けて見える。
- **中身 `coreMat`**: 光沢を抑えた（`clearcoat` 低め・`roughness` 高め）マット寄りに、
  ごく弱い自己発光 `emissive` を足して「現実にない謎の物体」感を出している。

## 調整パラメータ（`main.js` 冒頭 `CONFIG`）

ここを触れば見た目・挙動を変えられる。

### `CONFIG.shell`

| キー | 既定 | 意味 |
|---|---|---|
| `thickness` | 0.10 | 殻の不透明度（=見た目の厚み）。上げると厚いガラス |
| `softness`  | 0.0  | 0=剛体。上げると殻が柔らかく変形（接触処理も有効化） |
| `detail`    | 4    | 殻の分割数。大きいほど滑らか |

### `CONFIG.slime`

| キー | 既定 | 意味 / 触り方 |
|---|---|---|
| `detail`    | 4    | 中身の分割数。大きいほど均質で滑らか（重くなる） |
| `size`      | 3.5  | 中身の半径 |
| `pressure`  | 110  | 内圧＝張り。大きいほどプルッと丸く、小さいほどデロッと流れる |
| `stiffness` | 0.10 | バネの硬さ。小さいほど変形しやすく流れる |
| `damping`   | 0.99 | 速度減衰。1 に近いほどよく揺れる（プルプル） |
| `smooth`    | 0.40 | 表面のしわ取り＝表面張力。大きいほど滑らかな膜 |
| `shape`     | 0.0  | 形状記憶。0=流体寄り、大きいほど固体（丸へ戻る）寄り |

### その他

| キー | 既定 | 意味 |
|---|---|---|
| `CONFIG.gravity` | 4 | 重力の強さ（主に流れの速さに効く） |

### 味付けレシピ例

- **デロデロの水あめ風**: `pressure` 80, `stiffness` 0.1, `smooth` 0.3, `shape` 0
- **プルッと丸い水滴風**: `pressure` 200+, `smooth` 0.6, `shape` 0.01
- **ぷるぷるゼリー（液体と固体の中間）**: `pressure` 130, `shape` 0.04, `smooth` 0.45
- **しわが出る**: `smooth` を上げる / `pressure` を上げて表面を張らせる
- **揺れすぎて収束しない**: `damping` を下げる（例 0.985）

## 主要シンボル早見

| シンボル | 場所 | 役割 |
|---|---|---|
| `cPos / cPrev / cFrc` | main.js | 中身の現在位置 / 前回位置 / 力（Float32Array） |
| `cRest0` | main.js | 形状記憶の戻り先（原点中心の元の丸い形） |
| `cEdges` | main.js | 中身の距離拘束（バネ）の一覧 |
| `cAdj`   | main.js | 平滑化用の隣接リスト（CSR） |
| `gTarget` | gravity.js | 入力からの目標重力ベクトル |
| `gCur`    | main.js | 実際に使う（追従後の）重力ベクトル |

接頭辞 `s` は殻（shell）、`c` は中身（core/slime）に対応する。
